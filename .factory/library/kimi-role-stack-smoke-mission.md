# Kimi Role Stack Smoke Mission

Use this after restarting Droid so the new project droids and SessionStart hook are loaded.

Goal:
Prove that a Kimi-only Droid stack can carry one small real DGDH mission with cleaner role shape and less drift than the default setup.

Read first:
- `AGENTS.md`
- `CURRENT.md`
- `MEMORY.md`
- `company-hq/ACTIVE-MISSION.md`
- `SOUL.md`
- `company-hq/souls/nerah.md`
- `company-hq/souls/eidan.md`
- `company-hq/souls/taren.md`

Mission:
Take one small but real operator or triad-path friction and improve it end to end.

Choose the highest-leverage small mountain from this family:
- one missing readiness or truth surface
- one boring operator CLI or API improvement
- one narrow manual-rescue reduction in triad closeout
- one missing piece of runtime or mission legibility that still costs David minutes

Role shape:
- Nerah cuts and re-scopes the mission
- Eidan carries implementation and commits real progress
- Taren reviews for Core versus Slop and keeps the mountain honest
- Nerah and Taren should use the first-principles mission-cutting frame before choosing or re-cutting the mountain

Planning requirements:
- 2 to 4 concrete features
- 2 to 3 milestones
- explicit validation at the end of every milestone
- targeted tests and touched-package typechecks first

Success:
- one real DGDH path becomes more boring, visible, or self-carrying
- the change lands as reviewable repo truth
- the report clearly separates real value from mere activity

Do not optimize for:
- big architecture
- decorative role lore
- broad unrelated cleanup
- activity that only looks ambitious
