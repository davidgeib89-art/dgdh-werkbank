# Triad Mission Loop V1

Status: dormant next-phase contract
Authority: David
Lane: triad governance mode
Mission ID: `triad-mission-loop-v1`

## Purpose

Cut the first real operating loop in which DGDH can carry one mission through
three distinct internal voices instead of one execution lane plus repeated David restating.

This contract is intentionally not the active proving phase yet.
It exists so the firm can grow from mission-cell proof into the first small
AI-firm operating organism as soon as `repeatable-live-mission-cell-proof-v1`
is honestly complete.

## Operating Shape

The triad loop should look like this:

1. David provides only the seed:
   mission, budget, blast radius, done-when truth, and real Type-1 boundaries
2. Nerah cuts the mission into a bounded operating packet:
   worker brief, reviewer brief, success criteria, escalation edges
3. Eidan carries execution:
   build, test, commit, and push until `doneWhen` or a true blocker
4. Taren reviews and decides:
   accept, request changes, patch the narrowest truthful fix, or escalate
5. David is only pulled back in for true one-way doors:
   `main`, deploy, rights, secrets, irreversible external effects, or strategy re-cut

## Bounded Goal

Prove one real triad mission loop on a bounded internal company-improvement target.

The point is not to simulate a company in prose.
The point is to make one mission travel through:

1. seed intake
2. CEO cut
3. worker execution
4. reviewer correction or promotion
5. final escalation only if a real Type-1 boundary remains

with less David supervision than today's single-lane mission autonomy mode.

## Strong Success

`strong success` means all of these are true:

- one mission entered as a bounded David seed instead of a long restating chat
- a CEO cut produced a real execution packet and a real review packet
- the worker lane carried the mission to a reviewable end state or a real blocker
- the reviewer lane made a real accept / changes-requested / narrow-patch decision
- David was only needed for a true Type-1 decision or final strategic truth
- the loop left durable repo truth that makes the next triad mission easier to run

## Truthful Partial

`truthful partial` means:

- the triad loop was entered for real
- at least two role boundaries were carried honestly
- the remaining blocker is narrow, reviewable, and clearly the next real cut
- the work did not collapse back into David micro-steering disguised as autonomy

## Hard Stop

`hard stop` means:

- the loop cannot continue without a true Type-1 decision
- the reviewer/runtime lane needed for the loop does not yet exist in an honest form
- or the next step would widen into a new orchestration platform instead of proving one bounded triad loop

## Guard Metrics

- no skipping review just because the worker branch is green
- no David micro-management inside normal Type-2 terrain
- no pretending a chat summary is the same thing as a real CEO cut
- no promoting to `main` without explicit review truth
- no new platform, provider, or agent swarm expansion as pseudo-progress

## Required Preconditions

This contract should only activate after:

- `repeatable-live-mission-cell-proof-v1` is honestly complete
- mission-cell continuity is repeatable enough that the worker lane can carry a bounded mission without heavy restating
- the next missing truth is really role-separation and review flow, not basic mission continuity

## First Candidate Deliverables

When this contract activates, the first real triad loop should push toward:

1. a bounded mission intake packet David can hand off in one shot
2. a CEO-cut artifact that produces explicit worker and reviewer packets
3. one runtime-visible reviewer state machine:
   `ready_to_build -> in_execution -> ready_for_review -> changes_requested -> ready_to_promote -> type1_escalation`
4. one real worker-to-reviewer handoff
5. one real reviewer verdict that changes what happens next
6. durable promotion truth about what is now triad-bound versus still single-lane/carrier-bound

## One sentence

Mission autonomy proves the firm can improve itself once; the triad loop proves the firm can start behaving like a small company.
