# Replay Eval Promotion V1

Status: active bounded operating contract
Authority: David
Lane: mission autonomy mode

## Purpose

Make self-improvement accumulate instead of merely producing activity.

## Required Loops

Every mission cell must keep these distinct:

1. Research loop
2. Change loop
3. Eval loop
4. Promotion loop

## Minimum Eval Truth

Before promotion, the cut should have the smallest truthful proof that matches its nature:

- targeted tests for touched logic
- prompt or wakeup context checks when runtime behavior depends on injected truth
- replay or runtime probes when the change claims a narrower or safer operating path
- exact git truth for what is local, committed, and pushed

## Promotion Rule

Promote only when:

- the gain is reviewable and durable
- the path can be reused without chat reconstruction
- the guard metrics still hold

Keep experimental when:

- the proof is one-off or local-only
- the gain is mostly narrative
- the lane still depends on fragile human restatement

## Default Targets

Promotion should land in the smallest truthful durable target, usually one of:

- mission contracts
- mission-cell contracts
- `CURRENT.md`
- `MEMORY.md`
- the minimal runtime seam that makes the truth actually reusable
